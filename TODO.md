🏗️ Project: The Blueprint Hub (Status: Prototype V1)

Concept: A centralized platform for Software Requirements & Architecture artifacts, bridging the gap between "Idea" and "Implementation".

🎯 Core Philosophy

Decoupling: แยก Document/Design ออกจาก Source Code เพื่อให้เรียนรู้ Architecture ได้โดยไม่ต้องยึดติดภาษา

Standardization: บังคับใช้โครงสร้างข้อมูลตามหลัก SDLC Phase 1 (Inception/Requirements)

Community-Driven: ระบบ Open Contribution ที่ให้ชุมชนช่วยกันปรับปรุง Requirement ให้สมบูรณ์

🛠 Key Features Implemented

Granular Data Structure: แบ่งข้อมูลเป็น 9 Standard Sections (Project Info, Problem, Scope, Stakeholders, FRs, NFRs, Constraints, Assumptions, Acceptance)

Structured Artifacts: รองรับการเก็บข้อมูลแบบ Text, List, และ Diagram (Mermaid.js rendering)

Versioning System: รองรับการเรียนรู้เป็นลำดับขั้น (V0.1 Idea -> V1.0 MVP)

Contribution Workflow: ระบบ Suggestion Mode สำหรับ Volunteer เพื่อเสนอการแก้ไขโดยไม่กระทบต้นฉบับ

Interactive UI: Accordion sections, Activity Heatmap, และ Real-time editing preview

📚 Software Engineering Principles Applied

Progressive Disclosure: การซ่อนรายละเอียด (Collapse sections) เพื่อให้ User focus สิ่งที่สนใจ

Separation of Concerns: การแยกส่วน UI (React Components) ออกจาก Data Structure (Interfaces)

Requirement Engineering: การบังคับใช้ Template มาตรฐาน (FR/NFR/Constraints) เพื่อฝึกกระบวนการคิดที่เป็นระบบ

🔜 Next Steps (Backlog)

Traceability Matrix: Linking Requirements to Tests/Code.

Diff View: Visualizing changes between versions.

Export SRS: Auto-generating professional documents.

AI Quality Check: Automated feedback on requirement ambiguity.
