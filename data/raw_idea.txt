มีหลายอย่างถูกต้อง เเละมีของใหม่ที่ฉันไม่เคยคิดเเละเธอคิดเเทนเป็นไอเดีย 

ฉันจะยกตัวอย่างสถานการณ์ ฉันมีปัญหา ฉันเป็น CS student อยากทำ project ไม่รู้จะเริ่มอย่างไร เเม้จะมี open-src มากมาย เเต่นั้นไม่ใช่จุดประสงค์ของฉัน สิ่งที่ต้องการคือ documentation สำหรับทำ project เพราะ cs student อย่างฉันคิดว่าไม่ควรจะผูก project ที่จะทำไว้กับ programming language ควรจะสามารถทำ project อะไรก็ตาม ตาม docs ที่เขียนตามเเบบ standard software engineering SDLC

จนเกิด platform นี้ขึ้นมา มันทำงานด้วยการให้ cs student เข้ามายัง platform เพื่อหา software project standard software engineering docs following SDLC ตอนเข้าถ้า software project ที่คนอื่นทำไว้เเล้วก็สามารถอ่าน docs เเล้วเริ่มทำตามได้เลย เเต่ถ้าไม่มีเลย ก็สามารถ prompt ตามที่ต้องการได้ นี่เเหละเป็นไอเดียฉัน โดยเมื่อ prompt เเล้ว generative ai LLM ก็จะสามารถ standard software engineering docs SDLC ตามไอเดียนั้น โดย docs ที่ได้จะเขียนตาม standard software engineering เพื่อให้ cs student ได้ฝึกเเละการเขียน docs ที่ถูกต้องได้ด้วย จาก code ที่ส่งให้จะเห็นว่าตอนนี้ฉันกำหนดให้ LLM generate response เเค่ไม่กี่อย่าง เพราะฉันจะเริ่มจากการ test ก่อนเเล้วค่อย scale ไปเรื่อยๆ เพิ่ม key เข้าไปเรื่อยๆ เพื่อให้ LLM generate standard SDLC ตามที่กำหนด key ไว้

สมมติตอนนี้เราได้เอกสารสำหรับการสร้าง software project เเล้วเเต่ยังขาด prototype หรือภาพรวมของระบบ เเละนี่เเหละที่จะลอง excalidraw MCP เข้ามาช่วยสร้าง system visualization ตามที่ LLM generate standard software engineering docs โดยฉันมีเเผนจะให้ LLM อ่าน standard SDLC docs เเละพิจารณาการสร้าง system visualization คร่าวๆ โดยไม่ต้องละเอียดขนาดนั้นขอเเค่เป็น prototype เเละสามารถดูเเละอ่านเพื่อการสื่อสารกับคนอื่นได้ 

เเละไอเดียเพิ่มเติมคือ คนที่ cs student ที่ prompt นี้อาจจะหรือไม่รู้ว่าสิ่งที่ LLM สร้างขึ้นมานั้นถูกหรือผิดอย่างไร หรือ cs student ยังไม่มีประสบการณ์มากพอที่จะ validation ได้การสามารถ published สิ่งที่ LLM generate ขึ้นมาให้คนใน platform เข้ามา review , feedback , comment , contribute ได้ว่าจริงๆ เเล้วจากไอเดียขึ้น standard SDLC จะต้องเป็นเเบบนั้นเเบบนี้

จนสุดท้ายก็จะกลับมาวน Loop เดิมขึ้น เมื่อมี new cs student เข้ามาใหม่ก็จะ scroll || search หา software project ว่ามีคนเคยทำ software project ที่ตัวเองสนใจกำลังจะทำเเล้วหรือยัง ถ้ามีก็สามารถเริ่มต้นจาก standard SDLC คนก่อนหน้าได้เลย เเต่ถ้าไม่มีก็เเค่ prompt เเล้วจะได้ docs SDLC ตามไอเดียของตนเอง

เพิ่มจากส่วนที่ฉันบอกเรื่องการ contribute สมมติสถานการณ์มีพี่ senior ที่ทำงานเเล้วมา contribute เพิ่ม new feature ให้กับ standard docs SDLC ใดๆ standard docs SDLC นั้นก็จะกลายเป็นอีก version คงจะสงสัยใช่ไหม ทำไมถึงต้องเป็นอีก version เเผนของฉันคือ เมื่อมี muli new cs student ที่เข้ามายัง platform เเล้วอยากทำ software project ที่มีคนเคย prompt , contribute เเล้ว new cs student เหล่านั้นก็สามารถเลือก version ได้ว่า ฉันอยากลองทำ software project นี้โดยสามารถเลือก version ที่ยังไม่มี feature นั้น feature นี้ได้

เช่น software project calculator version 1 feature + - * / , version 2 + - * /  ^ จะเห็นว่าจะมี operation หรือ newe feature เพื่อให้ new cs student เลือกได้ว่าอยากเริ่มทำตั้งเเต่ version standard docs SDLC ได้ เเละจะทำเเบบนี้เช่นกันถ้าเป็น software project อื่นๆ เช่น CMS , ERP , E-Com , E-Learn

เอาเเค่นี้ก่อนจริงๆ มีไอเดียอื่นๆ อีกเเต่ขี้เกียจพิมพ์เเล้ว 

เพิ่มเติมอยากเเนะนำอะไรเพิ่มฉันไหม ก็สามารถ feedback , review ข้อดีข้อเสียความเสี่ยงที่ฉันอาจจะยังไม่รู้เเต่คิดจะทำอย่างเดียว

จริงๆ ฉันมี web prototype เเล้วเเต่ยังไม่มี feature เรื่องการ prompt ได้เเล้วให้ LLM generate standard docs SDLC ฉันเลยมาวิธีการทำเรื่องนี้ผ่าน python ก่อน

โดย web prototype ฉันเขียนขึ้นมาด้วย pure typescript nextjs มีเเผนที่จะใช้ polyglot microservice architecture เพื่อให้สามารถใช้ python นี้เชื่อมกับ typescript nextjs ได้ เพราะคิดเกียจ migration จาก python to typescript กังวลว่า typescript จะมี library สำหรับทำเรื่องนี้ หรือถ้ามีก็อาจจะย้ายเป็น typescript ทั้งหมดเเต่ตอนนี้อยู่ในช่วง test เลยลองด้วย python ก่อน
